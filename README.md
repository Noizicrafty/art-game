# Art Game (Арт-игра)

Веб-игра для уроков рисования: учитель создает комнату, дети заходят из браузера по коду комнаты в одной сети и участвуют в раунде «запомни и нарисуй».

## Что сделано

- Серверная часть работает на **Python** (`server.py`).
- Приложение слушает `0.0.0.0:1919` (можно хостить на личном сервере).
- Дети подключаются с других устройств по IP сервера и коду комнаты.
- Добавлен `Dockerfile` для контейнерного запуска.

## Быстрый запуск (Python)

Требуется Python 3.10+.

```bash
python server.py
```

После старта откройте:

- Учитель: `http://<IP_СЕРВЕРА>:1919`
- Дети: тот же адрес с их устройств.

## Запуск через Docker

Собрать образ:

```bash
docker build -t art-game .
```

Запустить контейнер:

```bash
docker run --rm -p 1919:1919 --name art-game art-game
```

После запуска приложение доступно на:

- `http://localhost:1919` (на хост-машине)
- `http://<IP_СЕРВЕРА>:1919` (для устройств в сети)

## Сетевой доступ

Чтобы дети могли подключаться:

- Устройства должны видеть сервер по сети.
- Порт `1919/tcp` должен быть открыт в firewall/security group.

Пример (Ubuntu + UFW):

```bash
sudo ufw allow 1919/tcp
sudo ufw status
```

## Игровой процесс

1. Учитель выбирает роль «Учитель» и создает комнату.
2. Получает 4-значный код комнаты.
3. Дети выбирают роль «Ребенок», вводят имя и код комнаты.
4. Учитель запускает раунд.
5. Запоминание изображения → рисование → (в multi) голосование → результаты.

## API (кратко)

- `POST /api/rooms` — создать комнату
- `GET /api/rooms/:code` — получить состояние комнаты
- `POST /api/rooms/:code/join` — присоединить ребенка
- `DELETE /api/rooms/:code/kid/:name` — удалить ребенка из лобби
- `POST /api/rooms/:code/stage` — сменить этап
- `POST /api/rooms/:code/art` — отправить рисунок/загрузку
- `POST /api/rooms/:code/vote` — голос
- `POST /api/rooms/:code/results` — рассчитать итоги
