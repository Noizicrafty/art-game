# Art Game (Арт-игра)

Веб-игра для уроков рисования: учитель создает комнату, дети заходят из браузера по коду комнаты в одной локальной сети и участвуют в раунде «запомни и нарисуй».

## Возможности

- Учитель создает комнату и получает 4-значный код.
- Дети подключаются по коду комнаты и имени.
- Режимы:
  - один ребенок (`single`);
  - несколько детей (`multi`) с голосованием за лучший рисунок.
- Этапы игры:
  1. Лобби
  2. Запоминание картинки (2 минуты)
  3. Рисование (5 минут)
  4. Голосование (для multi)
  5. Результаты
- Встроенный canvas-редактор + возможность загрузки изображения.

## Технологии

- Node.js HTTP server (`server.js`)
- Vanilla JS фронтенд (`app.js`)
- Статическая раздача ассетов и API в одном процессе

## Требования

- Node.js 18+ (рекомендуется)
- Браузеры в одной сети с сервером

## Запуск на личном сервере (порт 1919)

1. Установите зависимости (если появятся в будущем):
   ```bash
   npm install
   ```
2. Запустите сервер:
   ```bash
   npm start
   ```
3. Приложение слушает `0.0.0.0:1919`.
4. Откройте в браузере учителя:
   - `http://<IP_ВАШЕГО_СЕРВЕРА>:1919`
5. Дети открывают тот же адрес со своих устройств в сети.

## Сетевой доступ

Чтобы дети могли подключиться:

- Убедитесь, что устройства в одной сети (или есть маршрутизация/VPN).
- Откройте порт `1919/TCP` в firewall сервера.
- Если сервер в облаке — откройте порт в security group.

Пример для Ubuntu (UFW):

```bash
sudo ufw allow 1919/tcp
sudo ufw status
```

## Как играть

1. Учитель: **Учитель** → выбрать режим и тему → **Создать комнату**.
2. Показать детям 4-значный код комнаты.
3. Дети: **Ребенок** → ввести имя и код → **Войти**.
4. Учитель видит лобби и запускает раунд.
5. После завершения выводятся результаты.

## API (кратко)

- `POST /api/rooms` — создать комнату
- `GET /api/rooms/:code` — состояние комнаты
- `POST /api/rooms/:code/join` — вход ребенка
- `DELETE /api/rooms/:code/kid/:name` — удалить ребенка
- `POST /api/rooms/:code/stage` — сменить этап
- `POST /api/rooms/:code/art` — отправить рисунок
- `POST /api/rooms/:code/vote` — проголосовать
- `POST /api/rooms/:code/results` — посчитать результаты

## Структура проекта

- `server.js` — HTTP сервер + API + раздача файлов
- `app.js` — клиентская логика игры
- `index.html` — HTML-шаблон
- `styles.css` — стили
- `*.png` — игровые ассеты

## Продакшен-рекомендации

- Запускать через `systemd`/`pm2`.
- Использовать reverse proxy (Nginx/Caddy) при публикации в интернет.
- Добавить HTTPS (Let's Encrypt), если доступ не только из локальной сети.
